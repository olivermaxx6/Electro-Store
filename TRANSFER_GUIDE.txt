# SPPIX Fix Scripts Transfer Guide
# ===============================

# Files created:
# 1. fix_admin_and_storefront.sh (6,965 bytes) - Main fix script
# 2. nginx_admin_fix.conf (8,664 bytes) - Updated nginx configuration
# 3. verify_database_api.sh (10,181 bytes) - Database and API verification script

# How to transfer to your Linux server:

# Method 1: Using SCP (from Windows PowerShell/CMD)
scp fix_admin_and_storefront.sh root@your-server-ip:/opt/sppix-store/
scp nginx_admin_fix.conf root@your-server-ip:/opt/sppix-store/
scp verify_database_api.sh root@your-server-ip:/opt/sppix-store/

# Method 2: Using WinSCP or FileZilla
# 1. Open WinSCP/FileZilla
# 2. Connect to your server
# 3. Navigate to /opt/sppix-store/
# 4. Upload all three files

# Method 3: Manual copy-paste via SSH
# 1. SSH into your server: ssh root@your-server-ip
# 2. Create each file: nano filename
# 3. Copy and paste content
# 4. Save and exit: Ctrl+X, Y, Enter

# After transferring, run these commands on your server:

# 1. Make scripts executable
chmod +x fix_admin_and_storefront.sh
chmod +x verify_database_api.sh

# 2. Run the main fix script
./fix_admin_and_storefront.sh

# 3. Update nginx configuration
sudo cp nginx_admin_fix.conf /etc/nginx/sites-available/sppix
sudo nginx -t
sudo systemctl reload nginx

# 4. Verify everything is working
./verify_database_api.sh

# Expected results:
# - Admin panel: https://sppix.com/admin (should load correctly)
# - Storefront: https://sppix.com (should show products from admin)
# - API endpoints: All working correctly
# - Database: Connected and populated with sample data

# If you encounter issues:
# 1. Check logs: tail -f /opt/sppix-store/Backend/logs/django.log
# 2. Check nginx: sudo tail -f /var/log/nginx/sppix_error.log
# 3. Test manually: curl https://sppix.com/api/public/products/
